Prioridade A — Essenciais (PoC mínimo)

Boot / integração com bootloader (BIOS/UEFI) — rotina de inicialização que recebe controle do GRUB/UEFI e carrega o kernel na memória.

Inicialização (init) / espaço usuário mínimo — mecanismo para iniciar o primeiro processo (PID 1) e montar o userland.

Gerenciamento de memória física (RAM) — descobrir RAM disponível, reservar áreas, alocar/free páginas físicas.

MMU / memória virtual — implementar mapeamento linear/por processos, tabelas de páginas, proteção de páginas.

Gerenciamento de processos (scheduler) — criar/terminar processos, contexto de troca (context switch) e política de escalonamento básica.

Syscall/API (interface usuário-kernel) — conjunto de chamadas para I/O, processos, arquivos, memória (POSIX-like mínimo).

Tratamento de interrupções (IDT/ISRs) e timers — receber IRQs do hardware, temporizadores do sistema (tick).

Driver de console / saída básica — printk/log em serial/console para debug e mensagens do kernel.

Driver de bloco básico (acesso ao disco) — leitura/gravação de blocos para carregar sistemas de arquivos / swap.

Sistema de arquivos simples + VFS — abstração de FS e pelo menos um FS real (ex.: FAT/ext2) para montar rootfs.

Modelo de drivers / subsistema de dispositivos — registrar dispositivos, managers básicos para drivers carregarem.

Tabela de descritores/handles / file-descriptor layer — lidar com arquivos, sockets, pipes a partir de userland.

Prioridade B — Recomendados para uso prático

Driver de rede (NIC) e pilha TCP/IP — ethernet + stack IP (IPv4) para comunicação básica.

Driver Wi-Fi + suporte a firmware / wpa_supplicant (userland) — integração com gerenciamento de redes sem fio.

Gerenciamento de memória avançado (swap, páginas COW) — swap em disco, copy-on-write, cache de página.

Drivers de armazenamento (SATA, NVMe, SCSI) — suporte a discos modernos e SSDs.

Drivers de USB / bus enumeration — detectar e usar dispositivos USB.

Módulos dinâmicos (loadable kernel modules) — carregar/descargar drivers em runtime.

Kernel preemptible / políticas de scheduler melhoradas — para responsividade e multiusuário.

APIs de sincronização (mutexes, semáforos, futexes) — primitives para concorrência segura.

Logs persistentes / journal — sistema de logging e modo de diagnóstico.

Gestão de energia / ACPI — suspender/retomar, states de energia.

Prioridade C — Avançados / produção / otimização

Segurança: SELinux/AppArmor / namespaces / cgroups — isolamento, políticas, QoS, containers.

Driver de GPU / aceleração (Mesa/Vulkan stack em userland) — para GUI e aceleração gráfica.

Sistema de arquivos avançado (ext4, btrfs, zfs) — journaling, snapshots, integridade.

Rede avançada (IPv6, QoS, TLS offload, netfilter) — firewall, NAT, tun/tap, desempenho.

Subsystems: audio (ALSA/ Pulse), input (evdev), drm) — experiência desktop completa.

Kernel crashdump / kdump / oops handling — gerar dumps para análise pós-crash.

Real-time features (PREEMPT_RT) — se precisar de latência determinística.

Hotplug / device manager (udev-like) — automação de devices em userland.

Tooling de desenvolvimento (kbuild, kconfig, cross-compile toolchain) — para manter o kernel e builds reproduzíveis.

Compatibilidade binária / camada POSIX completa — para rodar aplicações comuns sem porting.
